# 🚀 Hardcore P2P Streamer

Быстрый P2P стример экрана и звука на базе WebRTC с поддержкой аппаратного ускорения, удаленного управления и многоканального аудио-микшера. 

## 🔥 Главные фичи
* **Аппаратное кодирование:** Поддержка NVENC (NVIDIA), AMF (AMD) и QSV (Intel) для захвата экрана с нулевой нагрузкой на CPU.
* **C++ Аудио Микшер:** Самописная DLL (`ProcessAudioCapture.dll`) на базе WASAPI для захвата звука конкретных окон. Можно на лету включать/выключать звук отдельных программ (например, стримить игру, но скрыть звук Discord).
* **Горячий рестарт:** Смена разрешения, пресета и монитора "на лету" без разрыва соединения у зрителя.
* **Удаленное управление:** Зритель может управлять мышью и клавиатурой (передача координат и скан-кодов клавиш через WebRTC DataChannel.
* **Умный JS Клиент:** Автоматический реконнект, обход блокировок автоплея в браузере, защита от зависаний (Watchdog).

## 🙏 Благодарности (Credits)
Благодарность [tsubome](https://github.com/tsubome/ProcessAudioCapture) за оригинальную реализацию библиотеки `ProcessAudioCapture` (базирующуюся на Microsoft ApplicationLoopback Sample). 

В данном проекте оригинальная DLL была существенно модифицирована:
* Добавлена функция `PacDataCallback` для прямого перехвата «сырых» PCM-байтов в реальном времени.
* Настроена интеграция с Python `ctypes` для обхода записи в файл.
* Убраны утечки памяти при жестком рестарте потоков.
